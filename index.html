Q1. What is Express.js?

✔️Express.js is a minimal and flexible Node.js web application framework that provides a robust set of features for building web and mobile applications.

Q2. Express.js works on top of which JavaScript runtime?

✔️Express.js works on top of Node.js runtime.

Q3. Why do we use Express instead of the built-in http module?

✔️We use Express instead of the built-in http module because it provides a more robust and flexible framework for building web applications.

Q4. What is a port number in a server?

✔️A port number is a number that is used to identify a specific service on a computer or network.

Q5. What does this line of code do? app.listen(3000);

✔️This line of code starts the Express server and makes it listen on port 3000.

Q6. What is middleware in Express?

✔️Middleware is a function that has access to the request object (req), the response object (res), and the next middleware function in the application’s request-response cycle.

Q7. What is the default HTTP method used when you type a website URL in the browser?

✔️The default HTTP method used when you type a website URL in the browser is GET.

Q8. What do 'req' and 'res' stand for in Express?

✔️'req' stands for request and 'res' stands for response.

Section B: Understanding & Short Explanation

Q9. Explain the difference between app.get() and app.post().

✔️app.get() is used to handle GET requests, typically for retrieving data from the server. app.post() is used to handle POST requests, typically for submitting data to the server.

Q10. What will happen if you run a server without specifying a port?

✔️If you run a server without specifying a port,it will still start successfully, but it will listen on a randomly assigned available port (also known as an ephemeral port) chosen by the operating system.

Q13. Why is middleware important in real-world applications? Give one example.

✔️Middleware is important in real-world applications because it allows developers to add functionality to the request-response cycle without modifying the core logic of the application. For example, middleware can be used for authentication, logging, error handling, and more.

Q14. What happens if two applications try to use the same port number?

✔️If two applications try to use the same port number, the second application will not be able to start successfully, as the port is already in use by the first application.

Q16. What will this middleware do? app.use((req, res, next) => { console.log("Someone made a request"); next(); }); Explain step by step what happens.

step 1: Middleware registration – app.use() tells Express: "Use this function for all requests, no matter the HTTP method (GET, POST, etc.) or URL path."

step 2: Request arrives – When a client makes any request (e.g., GET /, POST /user, etc.), Express starts processing it through the middleware chain.

step 3: This function executes – Express calls your function with three objects.
req: the request object
res: the response object
next: a function to call when this middleware is done

step 4: Console log – The line console.log("Someone made a request"); runs, printing that message to your terminal/console.

step 4: next() is called – This tells Express: "I'm finished with my task. Please move to the next middleware or route handler."

step 6: Request continues – Express proceeds to the next matching middleware or finally to your route handler (like app.get("/", ...)).

Q17. What will happen if we remove next() from middleware?

✔️If we remove next() from middleware, the request will not proceed to the next middleware or route handler, and the client will not receive a response, resulting in a hanging request.
